# Markdown/TXT 极速批量转 DOCX 合并工具

这是一个基于 Python 和 Pandoc 的高性能文档构建工具。它的核心功能是将指定目录下及其子目录内的多个 Markdown (`.md`) 或纯文本 (`.txt`) 文件**极速合并**并逐文件夹转换生成排版美观、字体统一、没有多余空行的 DOCX 文档。

---

## 🚀 核心特性

- **多进程并发极速转换**：基于 `ProcessPoolExecutor` 实现的多核 CPU 真并行，跳过了 Python GIL 锁的限制。多个带数百个文件的巨型文件夹能够被并行秒切出成型的 DOCX 文档。
- **O(1) 级全图样式清扫**：跳过了耗时极大的 Python-Docx DOM 树重构，直接调用底层 `lxml` XPath 对原生 Word DOM 树做就地修改（In-place Mutations），对排版进行了极高效率的清扫。
- **全自动排版休整**
  - **消灭换行/纯空行**：深度扫描去除 `Shift+Enter` 的硬换行以及多余的 `Enter` 段落空行，避免粘贴文本常见的松散结构。
  - **连续空格压缩**：将排版错误的多个连续空格归一化。
  - **首尾原位对齐**：正文及其表格默认进行两端对齐 (Justify) 洗数。
  - **章节强制物理分页**：自动在拼接点注入 OpenXML `<w:br w:type="page"/>` 硬分页符，防止首尾黏连。
- **全文档统一字体**：支持通过参数对一切内建样式（包括代码块 `Verbatim Char` 等顽固属性）实行全局自定义字体族覆盖，仅在正文微量调整字号，避免破坏标题层级、角标、表格的相对大小特性。
- **多编码智能 Fallback 探查**：默认 UTF-8 编码，遇到不兼容的系统截断文本会自动平滑降级到 GBK、GB2312。

## 📦 环境依赖

请确保您的执行环境中拥有已激活的 Python 环境（3.7+），并且安装了以下核心依赖库：

```bash
pip install python-docx pypandoc pypandoc_binary
```

> **注**：本脚本强依赖 `pypandoc_binary` 的独立 Pandoc 引擎环境支持。即便你当前的系统里没有配置过正式版的 Pandoc，`pypandoc_binary` 也会提供内置的无缝转换核心。

## 🛠️ 使用指南

最简单的直接启动方式是在脚本同级目录下运行：

```bash
python merge_md_to_docx.py
```

默认行为会：
1. 以递归方式找出当前运行目录（含当前目录）所有含有 `.md` 文件的文件夹。
2. 按照字母顺序对每一个底层文件夹里所有的 `.md` 进行拼接。
3. 对每个文件夹同名导出一个成品 `文件夹名称.docx` 文件，生成完毕的 DOCX 就存放在原来的目录下。
4. 全篇默认套用微软 `Aptos` 字体，正文使用统一的 12 号字。

### 命令行参数 (CLI Arguments)

此工具为不同需求使用场景设计了诸多可选项：

#### 指定目标扫描路径 (`--dir`)
不想放在文件目录下执行？可以通过绝对或相对路径指定扫描起步点。
```bash
python merge_md_to_docx.py --dir "C:\MyDocs\ProjectX"
```

#### 修改全局字体 (`--font`) 与正文字号 (`--font-size`)
无论之前的文本是什么字体，它都会无条件全部替换。由于采用只读式覆盖策略，替换字体并不会改变原有的倾斜、加粗、脚注样式状态！
```bash
# 设为宋体，字号加大到 14 号（近似小四）
python merge_md_to_docx.py --font "SimSun" --font-size 14
```

#### 合并纯文本 TXT (`--ext`)
此工具支持所有兼容 Pandoc Parse 的流式文本，如果你手头上有需要合并的普通 txt，只需一键覆写指定类型：
```bash
python merge_md_to_docx.py --ext .txt
# 运行后会对原本按 md 找寻的逻辑变为对 txt 的寻找合并
```

#### 并发数约束 (`--workers`)
如果处理太快引起发热或卡顿，你可以通过该参数将并发线程数降低，例如强制缩减为单进程单线程排队生成：
```bash
python merge_md_to_docx.py --workers 1
```

#### 统一页边距 (`--margin`)
你可以给统一生成的所有文档的上下左右边距限定死尺度（默认值为标准版式的 1.27 厘米）。
```bash
# 修改页边距到 2 厘米
python merge_md_to_docx.py --margin 2.0
```

## 🧐 常见问题解答 Q&A

**Q: 为什么部分原始文件里的表格跨页后排版错位？**
**A**: 本脚本的后处理模块专注于对齐与字体的极致抹平，它只负责洗去表格里单元格多余的错位空行。但复杂的表头跨页等，受限于 Word OpenXML 渲染器的底层机制，建议你在生成完毕后的文档里根据样式做最终审阅。

**Q: 执行中提示 `[WinError 32] 另一个程序正在使用此文件`?**
**A**: 这是绝大多数批量生成最常见的中断错误。如果你的 Microsoft Word 正处于打开状态并且**正在阅读旧版**的 `.docx` 生成品，它会向系统申请一个文件资源独占锁。请在执行工具前，**关闭相关且同名的那个 Word 视窗对象**。

**Q: 终端或者控制台跑的时候，Pandoc 弹出了少量类似 `[WARNING] Could not convert TeX math...` 的黄字/红字报错？**
**A**: 这是完全正常的现象，请忽略它们（只要最终输出 `✓ 已生成` 即可）。Markdown 中的一些非标准扩展符号、复杂的系统级 LaTeX 公式片段或是反斜杠，在 Pandoc 强转为 Word DOCX 时无法 100% 映射，Pandoc 本身会抛出解析警告，但它具有强大的鲁棒性，会自动将其降级为普通文本或跳过，**不会中断或影响文档生成！**
